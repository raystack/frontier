## Adding a frontend convention

Create a new `.md` file in this directory, link it above. Keep each file focused on one topic. Examples of topics that may belong here as the codebase evolves:

- Component patterns (when to use server vs client components, hook conventions)
- API client usage (typed clients, error handling)
- State management (zustand/redux/context choice rules)
- Styling and theme tokens
- Testing (RTL, E2E, mocking)
- Accessibility minimums
